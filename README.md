# Задание 4. Адаптивное приложение на Flutter

Кроссплатформенное приложение с адаптивным интерфейсом (Android, Desktop/Windows, Web, iOS).
Вариант **8**, развивает музыкальный проект из задания 3 лабораторной работы 8

## Возможности

- **Авторизация** (email/пароль) с сохранением сессии в Hive.
- **Главный экран — карточки данных** + график (fl_chart, избранное/остальные).
- **Списки** треков с добавлением/удалением (свайп) и отметкой «избранное».
- **Детальный просмотр** элемента (отдельный маршрут).
- **Тёмная/светлая тема** (системная + ручное переключение, `ColorScheme.fromSeed`).
- **Офлайн-режим** — данные кэшируются в Hive и доступны без сети.
- **Уведомления** — Toast/SnackBar (кроссплатформенно).
- **Настройки** — тема, язык, очистка кэша, версия, выход.
- **Локализация ru / en / be** с переключением в интерфейсе.
- **Адаптивная вёрстка** — `NavigationBar` (узкие экраны) ↔ `NavigationRail` (широкие), `MediaQuery`/breakpoints.
- Обработка исключений (сеть) с выводом в консоль и офлайн-fallback.

## Технологии

- **Flutter 3.44 / Dart 3.12**.
- **flutter_riverpod** — управление состоянием.
- **go_router** — навигация и маршрутизация (в т.ч. URL на Web).
- **Hive** — локальное хранилище (сессия, кэш, данные).
- **Dio** — сеть (загрузка списка с публичного API без ключа).
- **fl_chart** — график на главном экране.
- **flutter_localizations + intl** — локализация (ARB `lib/l10n`).

## Сборка и запуск

```bash
flutter pub get
flutter run -d windows      # Desktop
flutter run -d chrome       # Web
flutter run -d <android>    # Android (эмулятор/устройство)
```

Вход: e-mail уже подставлен (`user@example.com`), введите любой пароль длиной ≥ 4 символов.

## Тесты

```bash
flutter test                                   # модульные + виджет-тесты
flutter test integration_test/app_test.dart    # интеграционный тест (нужен запущенный девайс/desktop)
```

- **Модульные** (`test/unit_test.dart`) — валидация входа, сериализация модели, логика `TracksNotifier`.
- **Виджет-тесты** (`test/widget_test.dart`) — экран входа, ошибка валидации, главный экран с данными.
- **Интеграционный** (`integration_test/app_test.dart`) — сквозной сценарий: вход → главный экран.
